Absolutely—let’s lock down those forms with CSRF protection in three steps:

1. Generate & store a token in session

In your private/config.php (or wherever you bootstrap sessions), ensure you start the session and create a token if one doesn’t exist yet. For example, at the very top:

php
<?php
session_start();

// Create a CSRF token if needed
if (empty($_SESSION['csrf_token'])) {
    // 32 bytes = 64 hex characters
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}

This guarantees every user session has a unique, unpredictable token.

2. Output the token in your forms

In each form (e.g. add-new-market.php, edit-vendor.php, etc.), add a hidden input carrying the token. You can either echo it inline, or—better—wrap it in a small helper in private/validation.php (or Utils) for reuse:

php
// In private/validation.php (or Utils.php)
function csrf_input_tag(): string {
    $token = $_SESSION['csrf_token'] ?? '';
    return "<input type=\"hidden\" name=\"csrf_token\" value=\"{$token}\">";
}

Then in your HTML form:

php
<form method="post" action="your-handler.php">
  <?php echo csrf_input_tag(); ?>

  <!-- other fields… -->
  <button type="submit">Save</button>
</form>

3. Verify the token on form submission

At the very top of your POST handler—before processing or sanitizing other inputs—check that the incoming token matches the session. If it doesn’t, reject the request:

php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    // 1) Check CSRF token
    $submitted = $_POST['csrf_token'] ?? '';
    if (
        empty($submitted) ||
        !hash_equals($_SESSION['csrf_token'], $submitted)
    ) {
        // Invalid token: stop here
        Utils::setFlashMessage('error', 'Invalid CSRF token.');
        header('Location: ' . $_SERVER['PHP_SELF']);
        exit;
    }

    // 2) (Optional) Regenerate token to prevent replay
    unset($_SESSION['csrf_token']);

    // 3) Sanitize & process your form as usual
    $_POST = Utils::sanitize($_POST);
    // …
}

Why hash_equals?
It prevents timing attacks compared to a straight === comparison.

    // …
